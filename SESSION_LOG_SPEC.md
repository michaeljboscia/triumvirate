# Cross-Agent Session Log Specification v1.0

**Purpose:** Define the authoritative format for session logs across Claude, Gemini, and Codex so all three agents produce compatible, mergeable session histories.

**Author:** Claude Opus 4.5 + Mike Boscia
**Date:** 2026-02-12

---

## 1. Naming Convention

### Format
```
<owner>--<client>_<domain>_<repo>_<feature>_<YYYYMMDD>_v<N>_<agent>.md
```

### Components

| Component | Description | Example |
|-----------|-------------|---------|
| `owner` | GitHub username or identifier | `michaeljboscia` |
| `client` | Project/client name | `gtm-machine` |
| `domain` | Work domain | `outreach`, `infrastructure`, `intelligence` |
| `repo` | Repository name | `gtm-machine-infrastructure` |
| `feature` | Feature/task being worked on | `narrative_generator` |
| `YYYYMMDD` | Date (no separators) | `20260212` |
| `v<N>` | Version number (increments within same day/feature) | `v1`, `v2`, `v3` |
| `agent` | Which AI produced this log | `claude`, `gemini`, `codex` |

### Examples
```
michaeljboscia--gtm-machine_outreach_gtm-machine-infrastructure_narrative_generator_20260212_v1_claude.md
michaeljboscia--gtm-machine_outreach_gtm-machine-infrastructure_narrative_generator_20260212_v2_gemini.md
michaeljboscia--gtm-machine_outreach_gtm-machine-infrastructure_narrative_generator_20260212_v3_codex.md
```

### Location
All session logs go to: `<project-root>/session-logs/`

If no project context, fall back to agent's home:
- Claude: `~/.claude/session-logs/`
- Gemini: `~/.gemini/session-logs/`
- Codex: `~/.codex/session-logs/`

## 3. Native Transcript References

Each agent stores raw transcripts differently. The session log must reference the native source:

| Agent | Native Transcript Location | Reference Format |
|-------|---------------------------|------------------|
| Claude | `~/.claude/projects/-Users-mikeboscia/<UUID>.jsonl` | UUID only: `72512663-f081-45d3-9ed7-dab84ac8bafb` |
| Gemini | (similar JSONL) | UUID or path |
| Codex | `~/.codex/sessions/YYYY/MM/DD/<rollout-timestamp-id>.jsonl` | Full relative path: `sessions/2026/02/12/rollout-2026-02-12T...` |

## 4. Taxonomy Source

Each project should have `.claude/taxonomy.json` (or equivalent) defining:

```json
{
  "owner": "michaeljboscia",
  "client": "gtm-machine",
  "domain": "outreach",
  "repo": "gtm-machine-infrastructure",
  "feature": "narrative_generator"
}
```

If missing, agents should:
1. Infer from git remote URL
2. Infer from directory name
3. Ask user
4. Use "unknown" as fallback

---

## 5. Version Numbering

Within the same day + feature + agent:
- First log: `v1`
- Each subsequent log: increment by 1

Across agents on same day/feature:
- Each agent maintains its own version counter
- `..._v1_claude.md`, `..._v1_gemini.md`, `..._v1_codex.md` can coexist

## 6. Git Workflow

### On Session Log Creation

```bash
cd <project-root>
git add session-logs/<new-log-file>.md
git commit -m "session(<agent>): <feature> v<N> - <brief description>"
git push origin <branch>
```

### Commit Message Format
```
session(<agent>): <feature> v<N> - <brief summary>

Examples:
session(claude): narrative_generator v3 - Fixed business_impact bug
session(gemini): pain_sensors v1 - Initial data collection
session(codex): competitor_analysis v2 - Added CrUX integration
```

## 7. Cross-Agent Compatibility

### Reading Sibling Logs
Any agent should be able to read and understand logs from siblings by:
1. Parsing the standard markdown structure
2. Using TRANSCRIPT HISTORY to find all related work
3. Following native transcript references if more detail needed

### Merging Context
When resuming work another agent started:
1. Read most recent log from ANY agent for this feature
2. Check TRANSCRIPT HISTORY for full context chain
3. Continue from CURRENT STATE section

## 8. Privacy/Redaction Rules

### Never Include in Session Logs
- Full API keys (show only last 4 chars: `...abc123`)
- OAuth tokens
- Passwords
- Personal data beyond what's necessary

### Safe to Include
- API endpoint URLs
- Project IDs (Supabase ref, etc.)
- File paths
- Commit hashes
- Error messages (redact any embedded secrets)

---

## 9. Retention Policy

### Keep Forever (Git-Tracked)
- Session log markdown files
- Taxonomy files

### Ephemeral (Not Git-Tracked)
- Native JSONL transcripts (agent-specific retention)
- Cache files
- Temp files

### Recommended Native Transcript Retention
- Keep for 30 days minimum
- Archive to cold storage after 90 days
- Delete after 1 year unless flagged for preservation

---

## 10. Recovery Behavior

### If Session Log is Corrupt/Missing
1. Check for native transcript reference
2. Regenerate summary from native transcript
3. If native also missing, mark gap in TRANSCRIPT HISTORY

### If Mid-Session Crash
- Agent should attempt to write partial log on recovery
- Mark with `**Status:** INCOMPLETE - session crashed`
- Include whatever context was captured

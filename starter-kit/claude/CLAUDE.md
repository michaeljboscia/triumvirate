## IRON LAW: Always Use Fully Qualified File Paths

**NEVER reference a file with a stub, relative, or abbreviated path.**

Every file reference — in responses, plans, tool calls, and documentation — MUST use the full absolute path.

**Why:** Relative paths fail after context compaction (working dir is lost), break in plan mode (restarts at `$HOME`), and fail silently in inter-agent messages.

---

## IRON LAW: Context Compaction Recovery

**When returning from context compaction: READ THE SESSION LOG FIRST.**

The hooks auto-save session state before compaction occurs. After compaction:
1. The `post-compact-recovery.sh` hook injects the summary automatically
2. If context is insufficient, find the latest session log:
   ```
   ls -t <project>/session-logs/*_v*.md | head -1
   ```
3. Read it — it contains your memory

### Session Log MUST Contain:
1. **OVERALL GOAL** — What are we building?
2. **WHERE WE ARE** — Current phase, current task
3. **WHAT WORKS** — Verified working components
4. **WHAT DOESN'T WORK** — Failures and root causes
5. **CRITICAL PATHS** — URLs, file paths, API keys
6. **SESSION ACTIVITY LOG** — Time-stamped actions

## IRON LAW: Commit Early, Commit Often

Every meaningful change goes to git. The `post-tool-use.sh` hook auto-stages
files after Edit/Write, but you should commit with descriptive messages at
natural checkpoints.

---

## IRON LAW: Lessons Are Captured in Real-Time

When you encounter something lesson-worthy (unexpected error, abandoned approach,
wrong assumption), add it to the project's `lessons.md` immediately.

Format:
```markdown
## YYYY-MM-DD — Short Title
💥 What happened: [incident]
✅ Lesson: [actionable takeaway]
Scope: project | global
```

**Gemini** → `mcp__inter-agent-gemini__spawn_daemon()` / `ask_daemon()` / `dismiss_daemon()`

### 3 Failures = Ask for Help

## Hook System Overview

The hooks provide an automated session lifecycle:

```
SessionStart → session-start.sh (project picker OR session recovery)
             → post-compact-recovery.sh (restore context after compaction)

Every Tool Call:
  PreToolUse  → pre-tool-use-artifact-guard.sh (The Airlock — snapshot files + Supabase gate)
              → pre-tool-use-bash-guard.sh (block destructive SQL without backup)
  [Tool runs]
  PostToolUse → post-tool-use.sh (auto-stage, log activity)
              → token-gate.sh (auto-save at ~50K token intervals)

Before Memory Loss:
  PreCompact  → pre-compact.sh (Gemini summarization → session log → git commit)
```

## Taxonomy System

Each project has a `.claude/taxonomy.json` that identifies it:

```json
{
  "owner": "your-username",
  "client": "client-name",
  "domain": "infrastructure",
  "repo": "project-name",
  "feature": "current-feature"
}
```

Session logs are named using this taxonomy:
`<owner>--<client>_<domain>_<repo>_<feature>_<YYYYMMDD>_v<N>_<agent>.md`

This naming convention is shared across Claude, Codex, and Gemini — any agent
can find and read any other agent's session logs.

## Environment Variables

| Variable | Purpose | Default |
|----------|---------|---------|
| `GEMINI_CLI_PATH` | Path to Gemini CLI binary | `gemini` (from PATH) |
| `AI_MEMORY_DIR` | Central session log store | `~/.ai-memory` |
| `SUPABASE_BACKUP_TTL_MINS` | Backup freshness for Supabase gate | `30` |
| `TOKEN_GATE_THRESHOLD_KB` | Transcript growth before auto-save | `200` |
| `TOKEN_GATE_COOLDOWN_SECS` | Min seconds between auto-saves | `300` |
| `ARTIFACT_GUARD_BYPASS` | Emergency bypass for the Airlock | `0` |
| `BASH_GUARD_BYPASS` | Emergency bypass for bash guard | `0` |

## Getting Started

1. Create a project: `mkdir -p ~/projects/my-project/.claude`
2. Add taxonomy: copy `taxonomy.json.example` to `~/projects/my-project/.claude/taxonomy.json`
3. Start Claude Code from the project: `cd ~/projects/my-project && claude`
4. The project picker will recognize the project automatically

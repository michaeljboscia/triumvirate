## Role in the Triumvirate

You are one of three AI coding assistants operating on the same infrastructure.
Session logs are shared memory — all three agents read/write the same format.

### Your Siblings
- **Claude Code** (`claude` command) — Primary development agent, runs hooks
- **Gemini** (`gemini` command) — Research, analysis, and long-running tasks

### Session Log Format
All agents use the same naming convention:
`<owner>--<client>_<domain>_<repo>_<feature>_<YYYYMMDD>_v<N>_<agent>.md`

Where `<agent>` is `claude`, `codex`, or `gemini`.

---

## Key Principles

1. **Always use fully qualified file paths** — No relative paths, no tilde shortcuts
2. **Read session logs first** — They contain shared context from all agents
3. **Commit to git** — Every meaningful change gets committed
4. **Log lessons** — Unexpected findings go in `lessons.md`

## Git Self-Management

Codex should manage its own git operations:
- Stage files after modifications

## Inter-Agent Communication

### Sending to Claude
Use the `send-to-claude` skill:
```
/send-to-claude "Your question here"
```

### Sending to Gemini
Use the inter-agent MCP server (if configured in config.toml):
```
mcp__inter-agent-gemini__spawn_daemon → ask_daemon → dismiss_daemon
```

### Receiving Messages
When Claude or Gemini sends you a task, the message includes:
- The question or task
- A session log reference (read it for full context)
- The working directory
